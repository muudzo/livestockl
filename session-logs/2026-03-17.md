## What We Did Well

1. **Parallel agent workflow.** 6 agents building the Go backend simultaneously. Each had clear package boundaries. This is the right way to build with AI.

2. **Chaos testing found a real bug.** Concurrent bid test exposed a TOCTOU race condition. Fixed with atomic `sync_listing_bid()` PostgreSQL function.

3. **The payment retry metric.** "50% first-attempt → 100% with retry" — genuinely compelling for Paynow. Tells a revenue recovery story.

4. **SRP refactoring after shipping.** Built fast, then cleaned. Buyer-agent went from 5 responsibilities to 1. Dashboard from 400 lines to 100. Right order.

5. **Paynow root cause analysis.** Proved it with 6 methods, found forum confirmation, identified Cloudflare, framed as actionable recommendation.

6. **Senior review + fix cycle.** 23 issues found and all 23 fixed in one session — including critical WebSocket impersonation bypass and password hash leak.

## Great Ideas

1. **Agentic commerce as Paynow pitch.** "Agents will be your biggest merchants" — gives Paynow a business reason to fix their API.

2. **Counter-thought workflow.** Pushing back before agreeing led to better decisions — dropped QA from 7 to 3 agents, chose Go over Node.

3. **Simulated Zimbabwe payment failure rates.** EcoCash 70%, OneMoney 60%, Card 80% — makes the demo credible even when simulated.

4. **Dutch design nav.** Removing icons, pure typographic hierarchy. Bold differentiation from template UIs.

## Pitfalls

1. **Credentials in plain text.** Paynow key + Supabase access token exposed in conversation and git history. Rotate both before sharing the repo.

2. **Go agents produced wrong table names.** `FROM livestock` instead of `FROM livestock_items`. Fix: give parallel agents the migration file as input, not descriptions.

3. **Edge Function auth took 3 cycles.** Went through anon key → user JWT → service role key. Decide auth strategy upfront, not trial-and-error.

4. **UI basics missed during building.** 44px touch targets, `$` vs `US$`, emojis — should have been right the first time. The audit was necessary because we rushed.

5. **No Go unit tests.** Only curl-based QA. No `_test.go` files. Next person to touch `PlaceBid` could reintroduce the race condition silently.

6. **Branch sprawl.** `main`, `agentic`, `agenticv2`, `go-backend` — four branches, no merge strategy. Needs reconciliation before presenting.
