## What We Did Well

1. **Root cause analysis on chat bug** — didn't just look at the frontend; traced the full flow from UI → hook → RLS policy and found the real blocker was a database policy, not a code issue
2. **PWA approach over Capacitor** — chose the path that ships immediately with zero cost, matching the Zimbabwe Android-dominant market. Capacitor can layer on later without code changes
3. **Workbox caching strategy was thoughtful** — CacheFirst for images (don't re-download livestock photos), NetworkFirst for API (always fresh data), long cache for fonts. Matches actual usage patterns
4. **Kept the user informed** about what needs manual action (SQL in dashboard) vs what's automatic

## Pitfalls

- **RLS policy file ≠ live database** — editing `rls_policies.sql` doesn't auto-deploy to Supabase. The SQL fix still needs to be run manually in the Dashboard SQL Editor. This is easy to forget and could lead to "I fixed it but it's still broken" confusion
- **Supabase CLI version gap** — running v2.75.0, latest is v2.84.2. The older version lacks `db execute` which would have let us apply the SQL fix from the terminal. Worth upgrading
- **Session log gap** — last session log was March 23. Two weeks of work on this branch without documentation. Need to be more consistent
