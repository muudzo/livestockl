## Detail ŌĆö Ask #2 (BillPay biller) shipped

Paynow asked us to expose two endpoints they can call when a buyer pays an
auction reference via the BillPay menu (USSD / EcoCash app / etc.). Spec was
explicit about three gaps in the first cut we shipped on 2026-05-08, all closed
in commit `0b75500`:

1. **PAY must accept `currency`** ŌĆö added, validated as USD-only, explicit 400
   with the supported list on anything else
2. **AUTH `name` must return buyer name (not lot title)** ŌĆö refactored the join
   to surface buyer's first+last name; lot title moved to `description`
3. **Must be two URLs, not one dispatcher** ŌĆö split into separate functions

**Live endpoints**:

- AUTH ŌĆö `https://hmeieslclzycyjjjflfh.supabase.co/functions/v1/billpay-biller-auth`
- PAY ŌĆö `https://hmeieslclzycyjjjflfh.supabase.co/functions/v1/billpay-biller-pay`

**Idempotency contract** (PAY): same `paynowReference` returns byte-identical
`200 Paid` on every retry. Different `paynowReference` against an already-settled
member returns `409 AlreadyPaid`. Enforced at two layers ŌĆö application
short-circuit + partial unique index on `payments.paynow_reference WHERE
method='BillPay'`.

**Auth**: HTTP Basic via `BILLPAY_BILLER_USERNAME` / `BILLPAY_BILLER_PASSWORD`
Supabase secrets, plus optional `BILLPAY_BILLER_ALLOWED_IPS` allowlist. Every
inbound call logged to `public.billpay_inbound_log` for reconciliation.

**Tooling shipped alongside**:

- Postman collection at `billpay-biller.postman_collection.json` ŌĆö pre-wired
  Basic auth, variables for `member`/`amount`/`paynowReference`, and saved
  example responses for every status (200 Authorized, 200 AlreadyPaid, 200 Paid,
  200 Paid idempotent retry, 400 amount mismatch, 400 unsupported currency, 400
  missing field, 404 NonExistent, 409 AlreadyPaid, 401 Unauthorized)
- Three integration reference docs in `deliverables/week-6/` covering Paynow
  Merchant Services, BillPay, and TXT/SMS with file:line code references
  written for Paynow senior engineers to audit in-tree

**What's left**: Paynow needs to register us in the BillPay vendor portal with
the two URLs above + the Basic auth creds.

## Engineering notes

- Initial deploy of `billpay-biller-auth` / `billpay-biller-pay` 401'd at
  Supabase's gateway with `UNAUTHORIZED_INVALID_JWT_FORMAT` ŌĆö the gateway wants
  a JWT by default, but biller endpoints use Basic auth from Paynow. Fixed by
  marking both functions `verify_jwt = false` in `supabase/config.toml` and
  redeploying with `--no-verify-jwt`. Pattern matches existing webhook-style
  functions (`payment-webhook`, `billpay-reconcile`, etc.).
