## Journey-by-Journey

### 1. HomeFeed `/`  — **FAIL (P0)**
- Loads in <1 s. Search bar, six category chips, services rail (airtime / ZESA / Test Biller / Water), stats card render.
- Stats show **0 Animals sold · 0 Live auctions · 8 Buyers & sellers**.
- Grid shows **"No listings found · Try adjusting your search or browse all categories"** with a `Clear filters` button — even though "All" is selected and search is empty.
- Root cause: `src/hooks/useLivestock.ts:16` only returns `mockLivestock` when `isSupabaseConfigured === false`. `.env.local` *is* configured, so the hook queries the real Supabase project (`hmeieslclzycyjjjflfh`) which currently has **no active listings**. There is no fallback when the query succeeds with `data: []`.
- Knock-on: `/item/:id` cannot be exercised via card click (no card to click).

### 2. Item detail `/item/:id`  — **NOT TESTABLE (blocked by #1)**
Programmatic click on `a[href^='/item/']` from `/` returned zero matches.

### 3. Auth `/auth`  — **PASS (with cosmetic notes)**
- Bold editorial design (Newsreader/Fraunces serifs, IBM Plex Mono accents, cream background, oxblood CTA). Distinctive and on-brand for the "Mimoo" identity.

## Cross-Cutting Findings

### P0 — Empty marketplace
`HomeFeed` returns nothing in production-config mode. **Fix options:** (a) seed the Supabase project with at least 10 demo listings; (b) add a fallback in `useLivestockList` to surface `mockLivestock` when the real query returns `[]` *and* no filters are applied, with a "Sample listings" banner; (c) hide the marketplace tab until inventory exists and route `/` to operators landing. Option (a) is the only one that doesn't paper over a problem.

### P1 — Brand fragmentation
Three names — *ZimLivestock* (project / repo / global instructions), *Livestock Marketplace* (Home title), *The Mimoo Livestock Catalogue* (auth, marketing). Either commit to "Mimoo" everywhere or strip it from `/auth`. Right now it looks like two products glued together.

### P1 — Tawk chat overlap on mobile
On 375 px the Tawk widget's "Hi! How can we help?" tooltip + "Ask a question / Tell me more" buttons overlap the empty-state copy in the centre of the screen. Either delay the proactive tooltip, anchor it below the fold, or hide on `/` empty state.

### P1 — Tawk loads on localhost
Console shows 9 CORS errors per page load for `embed.tawk.to/...` because the dev origin isn't allow-listed at Tawk. Cosmetic for users (Tawk is configured for `*.zimlivestock.*`), but it floods devtools and slows the dev loop. Guard the script injection behind `import.meta.env.PROD` (or a `VITE_TAWKTO_ALLOWED_ORIGINS` check).

### P2 — Desktop layout locks to mobile width
At 1280 px the home feed is rendered inside a ~440 px column centred on a sea of white. The site is mobile-first by design (correct for the market), but desktop should at least display a 2- or 3-up grid or a hero. Currently it looks like a misconfigured viewport.

### P2 — Empty-state language
"No listings found · Try adjusting your search or browse all categories" — but the user hasn't searched or filtered anything. Branch the empty-state copy: "We're stocking the yard. Check back shortly." with a CTA to the operators page or a notify-me form.

### P2 — `/ussd-simulator` behind auth
A simulator is by definition something you reach for *before* you have an account. Either move it under `/operators/` (public demo) or expose it from the auth-screen footer.

### P2 — Native HTML5 validation on `/auth`
The rest of the page uses bespoke typography; the validation balloon is the browser default. Add inline error messages styled to match.

---

## Top 5 Recommended Fixes (in priority order)

1. **Seed Supabase with real listings** *(or* add a `data.length === 0 && !filtered` fallback in `useLivestockList` to render mock data with a banner). Without this, every demo and every first-time visit is dead on arrival.
2. **Pick one brand name** across Home, Auth, Operators, and the meta title. Recommend "ZimLivestock" (matches repo, footer, internship documents) and either retire "Mimoo" or scope it to a sub-brand.
3. **Gate Tawk.to to production origins.** One-line guard kills 9 CORS errors per dev page-load and prevents the mobile overlap on the empty state.
4. **Move `/ussd-simulator` to public** (or expose a "Try USSD" link from `/operators`) so the simulator is reachable without an account.
5. **Desktop layout pass on `/`.** Add a `lg:` grid (2–3 cards across) and a hero strip so a desktop visitor doesn't see a half-empty mobile column.
